1. Introdução

O FlatPatternHighlight é um plugin NXOpen em C#/.NET para o Siemens NX 2512 que automatiza a análise de flat patterns (padrões planos) de peças de Sheet Metal. O plugin processa o padrão plano em 3 etapas sequenciais e produz um log diagnóstico detalhado, além de criar opcionalmente cotas PMI em cadeia entre as linhas de centro de dobra e o contorno externo verdadeiro da peça.

1.1 O Problema

Em peças de chapa dobrada (Sheet Metal), o flat pattern gerado pelo NX contém:

  • 74+ curvas externas (GetExteriorCurves()) que incluem recortes internos, alívios, notches e furos — não apenas o contorno externo real.
  • 14+ linhas de centro de dobra que precisam ser correlacionadas com o perímetro externo para determinar qual lado de cada dobra aponta para a borda da peça.
  • O processo manual — contar dobras e inspecionar visualmente o contorno — é sujeito a erro e inconsistente.

1.2 A Solução

O plugin resolve esse problema com 3 etapas automáticas e determinísticas:

  1. Filtragem do Perímetro Externo — reduz as curvas externas brutas ao contorno externo verdadeiro usando P/Invoke em libufun.dll (UF_MODL_ask_face_loops).
  2. Linhas de Centro de Dobra — enumera e destaca as linhas de centro de dobra (up e down).
  3. Proximidade Dobra→Perímetro — para cada dobra, encontra a curva de perímetro paralela mais próxima em cada lado, cria um log tabular e gera cotas PMI em cadeia.

1.3 Fluxo de Análise (3 Steps)

Main()
  │
  ├─ FindFlatPattern() ─── localiza a Feature FlatPattern
  │
  ├─ [STEP 1] HighlightOuterPerimeter()
  │   ├─ GetExteriorCurves()         → 74 curvas brutas
  │   ├─ FindFlatSolidBody()         → 3 estratégias de busca
  │   ├─ UF_MODL_ask_face_loops()    → P/Invoke: outer loops das faces
  │   └─ outerPerim                  → somente curvas do contorno externo verdadeiro
  │
  ├─ [STEP 2] HighlightBendCenterLines()
  │   ├─ GetBendUpCenterLines()      → linhas de centro (up)
  │   ├─ GetBendDownCenterLines()    → linhas de centro (down)
  │   └─ bendLines                   → lista completa
  │
  └─ [STEP 3] AnalyzeBendToPerimeter()
      ├─ DetectNormalAxis()          → plano do flat pattern (XY, XZ ou YZ)
      ├─ Bounding box das curvas de perímetro
      ├─ Para cada bend line:
      │   ├─ Direção + normal perpendicular
      │   ├─ Filtra curvas paralelas (dot > 0.95) com overlap
      │   ├─ nearest / farIdx / secondBestIdx (rastreio multi-candidato)
      │   └─ bboxDist = distância até a borda do bounding box
      ├─ [SUBPASSO] Lane Clustering
      │   └─ Agrupa dobras paralelas com overlap em "flanges" independentes
      └─ [SUBPASSO] Chain PMI Dimensions
          └─ Cria cotas PMI: borda → 1ª dobra → 2ª dobra → ...

2.2 Build

O projeto utiliza o SDK style .csproj com Microsoft.NET.Sdk. As referências às DLLs do NX são resolvidas via HintPath.

Via script PowerShell (recomendado):

.\build.ps1 -NxDir D:\NX2512 -Configuration Release

O script build.ps1 executa automaticamente:

  1. Verifica pré-requisitos (dotnet SDK, DLLs do NX)
  2. Copia NXSigningResource.res para o diretório do projeto
  3. Atualiza o HintPath no .csproj para o caminho do NX
  4. Executa dotnet build -c Release
  5. Assina a DLL com SignDotNet.exe
  6. Copia a DLL, o .men e o .rtb para o diretório de deploy padrão (%UGII_USER_DIR%\startup)

Via linha de comando direta:

dotnet build -c Release --nologo -v q
SignDotNet.exe bin\Release\FlatPatternHighlight.dll

3. Referência de Classes & Métodos

3.1 Classe HighlightFlatPattern

Namespace: FlatPatternHighlight
Assembly: FlatPatternHighlight.dll
Tipo: Classe pública estática

Classe principal do plugin. Contém toda a lógica de análise do flat pattern em 3 etapas, além dos métodos utilitários, P/Invoke para chamadas nativas e criação de cotas PMI. Não requer instância — todos os métodos são estáticos.

3.2 Main() — Entry Point

PUBLIC STATIC int Main(string[] args)

Descrição: Ponto de entrada principal. Chamado quando o usuário executa o plugin via Ctrl+U, menu ou ribbon. Orquestra as 3 etapas sequenciais de análise.

Fluxo interno:

  1. Obtém a sessão NX ativa (Session.GetSession()) e a UI
  2. Valida a existência de um work part ativo
  3. Localiza a feature FlatPattern via FindFlatPattern()
  4. Executa HighlightOuterPerimeter() (Step 1)
  5. Executa HighlightBendCenterLines() (Step 2)
  6. Executa AnalyzeBendToPerimeter() (Step 3)
  7. Remove highlights temporários
  8. Retorna 0 em caso de sucesso

Tratamento de erros: Captura NXException e Exception genérica, exibindo mensagem em NXMessageBox.

3.3 Step 1 — HighlightOuterPerimeter()

PRIVATE STATIC void HighlightOuterPerimeter(
    FlatPattern flatPattern,
    Part workPart,
    LogFile lw,
    out List<Curve> outerPerim
)

Descrição: Filtra as curvas externas brutas obtidas de GetExteriorCurves() para reter apenas o contorno externo verdadeiro, excluindo notches, recortes e furos.

Algoritmo:

  1. Obtém todas as curvas externas via flatPattern.GetExteriorCurves()
  2. Localiza o corpo sólido do flat pattern via FindFlatSolidBody() (3 estratégias)
  3. Para cada face planar do corpo, chama UF_MODL_ask_face_loops() (P/Invoke)
  4. Filtra apenas outer loops (tipo 1), descartando inner loops (tipo 2)
  5. Cruzam-se as tags das arestas dos outer loops com as curvas exteriores
  6. Fallback: se o corpo não for encontrado, usa todas as curvas exteriores

Parâmetros:

  • flatPattern — Feature FlatPattern a ser analisada
  • workPart — Part ativa
  • lw — LogFile do NX para diagnóstico
  • outerPerim — Lista de saída com as curvas do perímetro externo

3.4 Step 2 — HighlightBendCenterLines()

PRIVATE STATIC void HighlightBendCenterLines(
    FlatPattern flatPattern,
    LogFile lw,
    List<Curve> outerPerim,
    out List<Curve> bendLines
)

Descrição: Recupera e destaca as linhas de centro de dobra (bend up e bend down). As linhas são destacadas visualmente no NX e um atributo de usuário "FlatPatternHighlight" = "true" é anexado a cada uma.

Parâmetros:

  • flatPattern — Feature FlatPattern
  • lw — LogFile para saída de diagnóstico
  • outerPerim — Perímetro externo (usado para detectar artefatos)
  • bendLines — Lista de saída com todas as linhas de centro de dobra

Detalhes:

  • GetBendUpCenterLines() — linhas de centro de dobra para flanges que dobram para cima
  • GetBendDownCenterLines() — linhas de centro de dobra para flanges que dobram para baixo
  • Linhas cujo midpoint 3D está a < 0.5 mm do perímetro externo são puladas (artefatos)

3.5 Step 3 — AnalyzeBendToPerimeter()

PRIVATE STATIC void AnalyzeBendToPerimeter(
    List<Curve> bendLines,
    List<Curve> outerPerim,
    LogFile lw,
    Part workPart
)

Descrição: Para cada linha de centro de dobra, encontra a curva de perímetro paralela mais próxima em cada lado (direção normal e oposta). Produz um log tabular detalhado e cria cotas PMI em cadeia.

Detecção do plano do Flat Pattern

PRIVATE STATIC int DetectNormalAxis(
    List<Curve> bendLines,
    List<Curve> outerPerim
)

Mede o spread (amplitude) dos endpoints de todas as curvas nos eixos X, Y, Z. O eixo com a menor amplitude é a normal do plano do flat pattern. Retorna 0 (X-normal), 1 (Y-normal) ou 2 (Z-normal).

Rastreio de Múltiplos Candidatos por Lado

Para cada dobra, o algoritmo rastreia simultaneamente 6 índices por lado (A = nml+, B = nml-):

CampoPropósito
bestIdx / bestDistCurva paralela mais próxima (com overlap). Pode ser um notch fino colado à dobra.
secondBestIdx / secondBestDist2ª curva paralela mais próxima. Usada para detectar cutout: se best/secondBest < 0.3, a "mais próxima" é um recorte fino e a segunda é a real.
farIdx / farDistCurva paralela mais distante — normalmente a borda exterior real. Corrigido por SmallEdgeRatio.
nearIdx / nearDistCurva mais próxima sem filtro de paralelismo. Fallback para dobras diagonais.
bestLineIdx / bestLineDistMesmo que bestIdx, mas restrito a objetos Line (ignora Arc).
farLineIdx / farLineDistMesmo que farIdx, mas restrito a Line. Preferido na dimensão PMI.

Correção SmallEdgeRatio

Se o segmento mais distante (farIdx) tem comprimento < 50% do segmento mais longo (longestIdx), e a distância do mais longo é ≥ 50% da distância do mais distante, então farIdx é substituído por longestIdx. Isso evita que notches de canto pequenos (ex.: 3 mm de comprimento) sejam selecionados como borda exterior em vez do segmento de borda verdadeiro (ex.: 400 mm).

Promoção Arc → Line para Dobras Diagonais

Uma dobra é diagonal quando tem componente significativa em ambos os eixos UV (|dir.X| > 0.2 E |dir.Y| > 0.2). Essas dobras raramente têm uma curva de perímetro com paralelismo perfeito (|dot| ≥ 0.95). O algoritmo:

  1. Se bestIdx for Arc (fillet de canto), troca para bestLineIdx no mesmo lado
  2. Se nenhum candidato paralelo foi encontrado, usa nearIdx como último recurso

3.6 Criação de Cotas PMI

CreateChainDimensions()

PRIVATE STATIC int CreateChainDimensions(
    Part workPart,
    List<bendInfo> bendInfos,
    List<perimData> perimData,
    List<Curve> outerPerim,
    double bboxMinU, double bboxMinV, double bboxMaxU, double bboxMaxV,
    int uAxis, int vAxis, int normalAxis,
    LogFile lw
)

Descrição: Ponto de entrada do dimensionamento em cadeia. Agrupa dobras paralelas em grupos de direção, divide cada grupo em lanes independentes e delega a criação de cotas para CreateChainForGroup().

ClusterByRangeOverlap()

PRIVATE STATIC List<List<int>> ClusterByRangeOverlap(
    List<bendInfo> bendInfos,
    List<int> groupIdx,
    int uAxis, int vAxis
)

Descrição: Divide um grupo de dobras paralelas em lanes (abas/flanges independentes). Algoritmo em dois passos:

  1. Atribuição greedy: Projeta endpoints na direção de referência; coloca a dobra na primeira lane que tenha sobreposição de range E comprimento similar (≥ 70%).
  2. Consolidação: Funde lanes que se sobrepõem após extensões de range, até convergência.

Threshold: LaneLengthRatioThreshold = 0.7 — a dobra mais curta deve ter pelo menos 70% do comprimento da mais longa para pertencer à mesma lane.

CreateChainForGroup()

PRIVATE STATIC int CreateChainForGroup(
    Part workPart,
    List<bendInfo> bendInfos,
    List<int> groupIdx,
    List<perimData> perimData,
    List<Curve> outerPerim,
    double bboxMinU, double bboxMinV, double bboxMaxU, double bboxMaxV,
    int uAxis, int vAxis, int normalAxis,
    LogFile lw
)

Descrição: Para uma lane, projeta os midpoints na direção normal de referência, classifica cada dobra como lowSide ou highSide (baseado na proximidade aos extremos do bbox), ordena cada lado e chama CreateChainSide().

CreateChainSide()

PRIVATE STATIC int CreateChainSide(
    Part workPart,
    List<bendInfo> bendInfos,
    List<(int idx, double offset, bool flipped)> side,
    List<perimData> perimData,
    List<Curve> outerPerim,
    bool isLowSide, int normalAxis,
    double bboxMinU, double bboxMinV, double bboxMaxU, double bboxMaxV,
    LogFile lw
)

Descrição: Cria as cotas PMI para um único lado de uma lane. Estrutura da cadeia: 

[outer boundary] ←dim0→ [bend_0] ←dim1→ [bend_1] ←dim2→ ... ←dimN→ [bend_N]

O boundary (borda exterior) é sempre o segmento paralelo mais distante (farLineIdx preferencialmente, depois farIdx) no lado correto. A regra de lado é:

  • isLow=true, flipped=false → Lado B (nml−)
  • isLow=true, flipped=true → Lado A (nml+)
  • isLow=false, flipped=false → Lado A (nml+)
  • isLow=false, flipped=true → Lado B (nml−)

CreatePmiRapidDimension()

PRIVATE STATIC bool CreatePmiRapidDimension(
    Part workPart,
    Curve curveA, Point3d pickA,
    Curve curveB, Point3d midB,
    Point3d origin, int normalAxis,
    LogFile lw
)

Descrição: Cria uma cota PMI perpendicular entre dois objetos Curve usando PmiRapidDimensionBuilder com MeasurementMethod.Perpendicular. O NX calcula a distância perpendicular geométrica verdadeira.

Cascata de Fallback:

OrdemCaminhoCondição

2Indicator-line vermelha (cor 36) entre os pontos de pickErro 948802 (sem licença PMI/GD&T)

CreatePmiPointFallbackDimension()

PRIVATE STATIC bool CreatePmiPointFallbackDimension(

    LogFile lw

Descrição: Fallback extremo: cria dois Points auxiliares (blanked) e uma cota horizontal/vertical entre eles. A direção é escolhida pelo eixo de maior variação entre os pontos.

CreateChainOrigin()

PRIVATE STATIC Point3d CreateChainOrigin(

Descrição: Calcula a posição do texto da cota. As cotas são empilhadas fora do bbox com margem base + espaçamento por nível. A direção de empilhamento depende de qual eixo (U ou V) é dominante na medição.

FindFlatPatternView()

Descrição: Localiza a modeling view do flat pattern (nome contém "FLAT") para usar como contexto de view nas cotas PMI. Retorna a work view se não encontrar.

3.8 P/Invoke — UF_MODL (libufun.dll)

O plugin utiliza P/Invoke para chamar funções nativas da biblioteca libufun.dll do NX, necessárias porque a API gerenciada NXOpen não expõe topologia de loops (outer vs inner loops) das faces.

Função NativaAssinatura GerenciadaPropósito
UF_MODL_ask_face_loops extern int UF_MODL_ask_face_loops(Tag face, out IntPtr loopList) Obtém a lista de loops de uma face
UF_MODL_ask_loop_list_count extern int UF_MODL_ask_loop_list_count(IntPtr loopList, out int count) Conta loops na lista
UF_MODL_ask_loop_list_item extern int UF_MODL_ask_loop_list_item(IntPtr loopList, int index, out int type, out IntPtr edgeList) Recupera um loop específico (type 1 = outer, 2 = inner)
UF_MODL_ask_list_count extern int UF_MODL_ask_list_count(IntPtr list, out int count) Conta itens em lista genérica
UF_MODL_ask_list_item extern int UF_MODL_ask_list_item(IntPtr list, int index, out Tag tag) Obtém tag de item por índice
UF_MODL_delete_loop_list extern int UF_MODL_delete_loop_list(ref IntPtr loopList) Libera a memória alocada pela UF_MODL_ask_face_loops
Atenção: Chamadas P/Invoke devem ser envolvidas entre JAM.StartUFCall("FlatPatternHighlight") e JAM.EndUFCall() para garantir consistência do contexto NX.

4.3 Cotas PMI em Cadeia

O Step 3 cria opcionalmente cotas PMI em cadeia. A estrutura gerada é: 

[borda exterior] ←dim0→ [dobra_0] ←dim1→ [dobra_1] ←dim2→ ... ←dimN→ [dobra_N]

Para cada aba (lane), duas cadeias são criadas — uma para o lado low e outra para o lado high.

Exemplo visual da cadeia em uma aba simples:

  ┌─────────────────────────────────────────────────┐
  │                  BORDA EXTERIOR                  │
  │                      |                           │
  │                  dim0 (29.85 mm)                 │
  │                      |                           │
  │              ┌─────── dobra_0 ───────┐           │
  │              |                        |          │

5. Algoritmos em Detalhe

5.1 Outer Perimeter Filtering

O algoritmo de filtragem do perímetro externo resolve o problema de que GetExteriorCurves() retorna todas as curvas da borda do flat pattern, incluindo contornos de notches, cutouts, alívios e furos. Apenas as curvas que pertencem a outer loops (loops externos) das faces planares do corpo sólido devem ser consideradas como "verdadeiro perímetro externo".

Estratégia:

  1. Localiza o corpo sólido do flat pattern via FindFlatSolidBody() (3 estratégias)
  2. Para cada face planar do corpo, chama UF_MODL_ask_face_loops() (P/Invoke)
  3. Filtra apenas loops do tipo 1 (outer loop). Loops tipo 2 (inner loop) são descartados
  4. Coleciona as tags de todas as arestas dos outer loops em um HashSet<Tag>
  5. Cruza as tags com as curvas exteriores: se ed.FlatSolidObject.Tag está no set, a curva é mantida
  6. Se o corpo não for encontrado, usa todas as curvas (fallback)

5.2 Bend-to-Perimeter Proximity

Para cada linha de centro de dobra, o algoritmo realiza as seguintes operações:

  1. Detecta o plano do flat pattern via DetectNormalAxis() — mede o spread dos endpoints em cada eixo; o eixo de menor spread é a normal.
  2. Calcula o bounding box 2D no plano UV (coordenadas do flat pattern).
  3. Para cada dobra:
    1. Calcula direção unitária (dir) e normal perpendicular (nml) no plano UV.
    2. Pula artefatos: midpoints 3D a < 0.5 mm do perímetro.
    3. Scan em todas as curvas de perímetro: calcula projeção na normal para determinar lado (A = nml+, B = nml-).
    4. Aplica filtro de paralelismo (|dot| ≥ 0.95) + filtro de overlap.
    5. Rastreia 6 candidatos por lado simultaneamente.
    6. Aplica correção SmallEdgeRatio.
    7. Para dobras diagonais, aplica promoção Arc→Line.
    8. Calcula distância ao bbox via ray-casting.

5.3 Lane Clustering

Problema: Uma peça pode ter múltiplas abas paralelas (ex.: abas esquerda e direita de uma dobra em U). Cada aba precisa de sua própria cadeia de cotas.

Solução: Algoritmo de clustering em 2 passos:

  1. Atribuição greedy:
    • Projeta os endpoints de cada dobra na direção de referência do grupo
    • Obtém o intervalo [lo, hi] da projeção
    • Coloca a dobra na primeira lane que tenha:
      • Sobreposição de range com [lo, hi]
      • Comprimento similar (ratio ≥ 0.7)
    • Se nenhuma lane combinar, cria uma nova lane
  2. Consolidação: Funde lanes que se sobrepõem após extensões de range, até convergência.

5.4 Chain PMI Dimensioning

A criação de cotas PMI segue uma hierarquia de decisões:

  1. Agrupa dobras paralelas em grupos de direção (|dot| ≥ 0.95).
  2. Divide cada grupo em lanes via ClusterByRangeOverlap().
  3. Para cada lane:
    1. Classifica dobras em lowSide e highSide pela projeção na normal de referência.
    2. Ordena lowSide ASC, highSide DESC.
    3. Para cada lado, chama CreateChainSide().
  4. CreateChainSide():
    1. Seleciona boundary (borda exterior) pelo farLineIdx/farIdx do lado correto.
    2. Cria cota boundary → primeira dobra.
    3. Cria cotas entre dobras consecutivas.
  5. Cada cota é criada via PmiRapidDimensionBuilder com fallbacks para indicator-line e point-fallback.

Seleção do Boundary — Regra de Lado:

6. Limitações Conhecidas

⚠ Importante: As limitações abaixo são conhecidas e algumas estão em análise para versões futuras.
LimitaçãoDescriçãoImpacto
Flat solid body não localizado Em algumas peças, GetBodies() e GetEntities() retornam null/vazio. A 3ª estratégia (busca exaustiva) pode falhar se a feature FlatSolid não estiver exposta. Fallback: todas as curvas externas são usadas (incluindo notches).
Artefatos em GetBendCenterLines Algumas configurações de flat pattern retornam arestas de perímetro através das APIs de bend center lines. Heurística de skip por midpoint (0.5 mm do perímetro) resolve a maioria dos casos.
PmiRapidDimension exige Line PmiRapidDimensionBuilder com MeasurementMethod.Perpendicular exige uma Line como 1ª referência. Arcs causam erro NX 1175009. Indicator-line vermelha é usada como fallback (sem cota numérica).
Licença PMI/GD&T Sem a licença, o builder retorna erro 948802. Indicator-lines vermelhas substituem as cotas.
Dobras diagonais Dobras com componente significativa em ambos os eixos UV têm poucos candidatos paralelos. Fallback para nearIdx (curva mais próxima sem filtro de paralelismo).
Cobertura de curvas não-Line/Arc O método GetEndPoints() retorna (0,0,0) para tipos de curva não suportados. Essas curvas são ignoradas na análise de proximidade.
